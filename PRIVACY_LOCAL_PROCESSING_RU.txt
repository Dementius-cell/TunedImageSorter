Tuned Image Sorter v69.6 / Этап 055 — privacy / local processing

Коротко:
  Приложение предназначено для локальной обработки папок с фотографиями и сортировки результата по распознанным лицам.

Что важно объяснить пользователю:

1. Исходные фото
   - Приложение читает выбранную input-папку.
   - Результаты создаются в выбранной output-папке.
   - Для безопасной проверки сначала используйте копию небольшой папки.

2. Результаты
   - Основные результаты находятся в папках people/final/review/reports внутри output.
   - Diagnostics и bug-report могут содержать техническую информацию о путях, runtime, ошибках и списках проблемных файлов.
   - Support-bundle не должен намеренно включать исходные фотографии.

3. Облако и сеть
   - Ordinary Start pipeline не требует отправки фотографий в облако.
   - GPU_FULL использует bundled CUDA runtime.
   - GPU_LITE может выполнять runtime setup в локальную папку пользователя; это относится к runtime-зависимостям, а не к отправке фото.

4. Чувствительные фото
   - Не используйте единственную копию важных фотографий как тестовую input-папку.
   - Проверяйте результат перед передачей другому человеку.
   - Удалите output-папку, если результат больше не нужен.

Known non-blocking UX issue:
  После обычного запуска кнопкой Старт папка reports может не открываться автоматически. Используйте кнопки открытия результата/отчётов в UI. Это не меняет результаты обработки.

Что не менялось:
  ML/recognition, clustering, ordinary Start pipeline, apply-names, schemas, result-health, support-bundle и GPU Lite runtime setup logic не менялись.
