Tuned Image Sorter v69.6 — RC checklist before sharing the package

Purpose: quickly verify that the portable package can be handed to another person.
This is not a new app mode and not a new pipeline. It is the final release-candidate checklist.

1. Check the version
   Open VERSION.txt.
   It must contain:
     version=v69.6
     refactor_stage=Этап 055
     ui_api_version=21

2. Check the portable package contents
   The following files must be next to TunedImageSorter.exe:
     START_HERE_RU.txt / START_HERE_EN.txt
     QUICK_START_RU.txt / QUICK_START_EN.txt
     FIRST_RUN_RU.txt / FIRST_RUN_EN.txt
     ERRORS_RU.txt / ERRORS_EN.txt
     TROUBLESHOOTING_RU.txt / TROUBLESHOOTING_EN.txt
     RC_CHECKLIST_RU.txt / RC_CHECKLIST_EN.txt
     RELEASE_GATE_RU.txt / RELEASE_GATE_EN.txt
     README_RU.txt / README_EN.txt
     VERSION.txt
     SUPPORT_BUNDLE_RU.txt / SUPPORT_BUNDLE_EN.txt
     portable_manifest.json
     package_identity_check.json / package_identity_check.txt

3. Check the CPU build
   Expected:
     TunedImageSorter_CPU
     TunedImageSorter_CPU_portable_v69_6.zip
     onnxruntime==1.27.0
     CUDAExecutionProvider is absent
     onnxruntime-gpu is not installed
     package_identity_check: OK
     friend_ready_package: OK
     zip_integrity: OK

4. Check the GPU build
   Expected:
     TunedImageSorter_GPU_FULL
     TunedImageSorter_GPU_FULL_portable_v69_6.zip
     onnxruntime-gpu==1.26.0
     CUDAExecutionProvider is visible
     CUDA session smoke-test: OK
     CUDA/cuDNN/cuBLAS runtime DLLs are present
     package_identity_check: OK
     friend_ready_package: OK
     zip_integrity: OK

5. Check release diagnostics
   Commands:
     TunedImageSorter_CLI.exe --release-check
     TunedImageSorter_CLI.exe --runtime-preflight
     TunedImageSorter_CLI.exe --runtime-preflight --gpu   # GPU package only

   Release-check must report OK.
   v69.6 must pass this guard:
     release_candidate_final_gate

6. Check the ordinary launch
   TunedImageSorter.exe must open the GUI without a persistent black console window.
   Ordinary Start must run the ordinary pipeline mode=all.
   Ordinary Start must not run apply-names.

7. Check a small photo test result
   For a small photo set:
     reports should be created
     diagnostics should open
     result-health should work
     support-bundle should be created

8. Check allowed warnings
   These are allowed if release-check and CUDAExecutionProvider are OK:
     scipy.special._cdflib not found
     nvinfer_10.dll / nvonnxparser_10.dll for TensorRT

9. Decision
   If all items above are OK, the build can be treated as an RC package that is ready to hand to a normal user.
   If any item is not OK, do not share the ZIP as stable; attach the support-bundle and build log.
